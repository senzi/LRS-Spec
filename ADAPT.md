# LRS (LLM Reading Stream) 适配提示词

> **任务**：请为我的当前项目适配 **LRS (LLM Reading Stream) 协议**，新增一个路由端点 `/lrs.md`。
> **核心逻辑**：
> 1. **根路径自描述 (必须)**：当**无参数**访问 `/lrs.md` 时，服务端 **必须** 直接返回本站点的 `SKILL.md` (能力说明书)。该说明书应包含本站点的参数用法及内容索引，以便 Agent 进一步抓取。注：本站 `SKILL.md` 通常可通过 `/lrs.md?slug=skill` 访问。
> 2. **预检模式 (`countOnly=true`)**：返回 JSON 格式的统计信息，包含 `postCount` (文章总数) 和 `totalChars` (正文总字符数)。支持配合 `year` 和 `month` 进行分段预检。
> 3. **时间切片查询**：支持 `year=YYYY` 和 `month=MM` 参数组合，用于过滤特定时间段的内容流，防止一次性加载过多内容。
> 4. **单篇模式 (`slug=[id]`)**：精确匹配并返回指定 ID 或 Slug 的单篇纯 Markdown 内容。
> 5. **列表模式 (`limit=N`)**：返回最近 N 篇文章的 Markdown 数据流。
> 
> 
> **技术规范**：
> * **响应头**：必须设置为 `text/markdown; charset=utf-8`。
> * **内容结构**：每篇文章必须包含 YAML Frontmatter 元数据，文章间用 `\n\n---\n\n` 分隔。
> * **路径补全**：请自动获取当前站点域名，并将正文中所有相对路径（图片、链接）转换为**绝对路径**。
> 
> 
> 请根据我当前项目的技术栈，为我生成该路由的实现代码。
